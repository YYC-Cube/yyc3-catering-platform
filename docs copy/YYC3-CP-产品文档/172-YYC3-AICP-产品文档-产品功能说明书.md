## 1. 产品概述

### 1.1 产品简介

YYC³-AICP (YanYuCloudCube AI Catering Platform) 是一款面向餐饮行业的智能化综合管理平台，通过多端协同、数据驱动、AI赋能的方式，为餐饮企业提供从订单管理、库存控制、员工调度到财务分析的全链路数字化解决方案。

### 1.2 产品定位

- **目标用户**：中小型餐饮企业、连锁餐厅、快餐店、咖啡厅等
- **核心价值**：提升运营效率、降低人力成本、优化顾客体验、数据驱动决策
- **产品特色**：多端协同、智能调度、实时监控、数据分析

### 1.3 产品架构

## 3. 功能详细说明

### 3.1 仪表盘功能

#### 3.1.1 数据概览

**功能描述**：实时展示餐厅核心经营数据，包括营业额、订单量、客流量、客单价等关键指标。

**数据指标**：
- 今日营业额：当日所有订单的总金额
- 今日订单量：当日所有订单的总数量
- 今日客流量：当日到店或下单的顾客数量
- 今日客单价：今日营业额 / 今日客流量
- 本月营业额：当月所有订单的总金额
- 本月订单量：当月所有订单的总数量
- 同比增长：与去年同期相比的增长率
- 环比增长：与上月相比的增长率

**数据更新频率**：实时更新（每5秒刷新一次）

**使用场景**：
- 管理者每日查看经营状况
- 分析营业趋势，制定经营策略
- 监控关键指标，及时发现问题

#### 3.1.2 趋势分析

**功能描述**：通过图表展示销售趋势、菜品销量趋势等数据，帮助管理者了解业务发展情况。

**图表类型**：
- 折线图：展示营业额、订单量随时间的变化趋势
- 柱状图：展示不同时间段的营业额、订单量对比
- 饼图：展示菜品分类占比、支付方式占比等
- 热力图：展示不同时间段的客流分布

**时间维度**：
- 日维度：展示每小时的数据变化
- 周维度：展示每天的数据变化
- 月维度：展示每周的数据变化
- 年维度：展示每月的数据变化

**使用场景**：
- 分析营业高峰时段，优化排班
- 了解菜品销量趋势，调整菜单
- 分析支付方式占比，优化支付配置

### 3.2 订单管理功能

#### 3.2.1 订单列表

**功能描述**：展示所有订单，支持筛选、搜索、导出等功能。

**列表字段**：
- 订单号：唯一标识订单的编号
- 订单时间：订单创建时间
- 顾客信息：顾客姓名、联系方式
- 订单金额：订单总金额
- 订单状态：待接单、制作中、配送中、已完成、已取消
- 支付状态：待支付、已支付、已退款
- 订单类型：堂食、外卖、自提

**筛选条件**：
- 时间范围：选择订单的时间范围
- 订单状态：选择订单状态进行筛选
- 订单类型：选择订单类型进行筛选
- 支付状态：选择支付状态进行筛选

**搜索功能**：
- 支持按订单号、顾客姓名、联系方式搜索

**导出功能**：
- 支持导出为Excel格式
- 支持导出为CSV格式
- 支持选择导出字段

#### 3.2.2 订单详情

**功能描述**：查看订单详细信息，包括菜品信息、金额明细、订单日志等。

**详情内容**：
- 订单基本信息：订单号、订单时间、顾客信息、订单类型等
- 菜品信息：菜品名称、数量、单价、小计
- 金额明细：菜品金额、配送费、优惠金额、实付金额
- 支付信息：支付方式、支付时间、支付状态
- 配送信息：配送地址、配送时间、配送员信息
- 订单日志：记录订单状态变更历史

**操作功能**：
- 打印订单：打印订单小票
- 修改订单：修改订单信息（特定状态下）
- 取消订单：取消订单（特定状态下）
- 退款：处理退款（特定状态下）

#### 3.2.3 订单处理

**功能描述**：处理订单状态变更，包括接单、制作、配送、完成等。

**状态流转**：
```
待接单 → 制作中 → 配送中 → 已完成
  ↓        ↓        ↓
已取消   已取消   已取消
```

**状态说明**：
- 待接单：订单已创建，等待接单
- 制作中：订单已接单，正在制作
- 配送中：订单制作完成，正在配送
- 已完成：订单已配送完成或顾客已取餐
- 已取消：订单已取消

**操作权限**：
- 接单：所有员工均可操作
- 制作：后厨员工可操作
- 配送：配送员可操作
- 完成：配送员或收银员可操作
- 取消：管理员可操作

### 3.4 库存管理功能

#### 3.4.1 库存列表

**功能描述**：查看所有库存物品，包括原料、半成品等。

**列表字段**：
- 物品名称：库存物品名称
- 物品分类：库存物品分类
- 当前库存：当前库存数量
- 单位：库存单位
- 预警阈值：库存预警阈值
- 状态：正常、预警、缺货
- 最后更新：最后更新时间

**状态说明**：
- 正常：库存数量 ≥ 预警阈值
- 预警：库存数量 < 预警阈值，但 > 0
- 缺货：库存数量 = 0

#### 3.4.2 库存预警

**功能描述**：设置库存预警阈值，自动提醒补货。

**预警规则**：
- 当库存数量低于预警阈值时，自动发送预警通知
- 预警通知方式：站内信、短信、邮件
- 预警频率：每天最多发送一次

**预警设置**：
- 设置预警阈值
- 选择预警通知方式
- 设置预警接收人

#### 3.4.3 库存盘点

**功能描述**：执行库存盘点，记录实际库存数量。

**盘点流程**：
1. 创建盘点单
2. 录入实际库存数量
3. 生成盘点差异报告
4. 确认盘点结果
5. 更新库存数量

**盘点差异**：
- 盘盈：实际库存 > 账面库存
- 盘亏：实际库存 < 账面库存

## 4. 操作流程与规范

### 4.1 订单处理流程

#### 4.1.1 堂食订单流程

```
顾客到店 → 扫码点餐 → 提交订单 → 支付订单 → 接单 → 制作 → 上菜 → 结账 → 完成
```

**操作规范**：
1. 顾客到店后，引导顾客扫码点餐
2. 顾客提交订单后，及时接单
3. 接单后，通知后厨开始制作
4. 制作完成后，及时上菜
5. 用餐结束后，引导顾客结账
6. 确认支付后，标记订单完成

#### 4.1.2 外卖订单流程

```
顾客下单 → 支付订单 → 接单 → 制作 → 配送 → 完成 → 评价
```

**操作规范**：
1. 顾客下单后，及时接单
2. 接单后，通知后厨开始制作
3. 制作完成后，通知配送员取餐
4. 配送员取餐后，开始配送
5. 配送完成后，标记订单完成
6. 引导顾客评价

#### 4.1.3 自提订单流程

```
顾客下单 → 支付订单 → 接单 → 制作 → 顾客取餐 → 完成
```

**操作规范**：
1. 顾客下单后，及时接单
2. 接单后，通知后厨开始制作
3. 制作完成后，通知顾客取餐
4. 顾客取餐后，标记订单完成

---

### 4.2 库存管理流程

#### 4.2.1 采购流程

```
库存预警 → 生成采购单 → 审核采购单 → 采购 → 入库 → 更新库存
```

**操作规范**：
1. 收到库存预警后，生成采购单
2. 审核采购单，确认采购数量和金额
3. 执行采购，联系供应商
4. 采购完成后，验收入库
5. 更新库存数量

#### 4.2.2 盘点流程

```
创建盘点单 → 录入实际库存 → 生成差异报告 → 确认盘点结果 → 更新库存
```

**操作规范**：
1. 创建盘点单，设置盘点范围
2. 录入实际库存数量
3. 生成盘点差异报告
4. 审核差异报告，确认盘点结果
5. 更新库存数量

---

### 4.3 员工管理流程

#### 4.3.1 入职流程

```
面试 → 录用 → 办理入职 → 培训 → 上岗
```

**操作规范**：
1. 面试合格后，办理录用手续
2. 填写入职登记表，收集个人信息
3. 安排培训，包括产品培训、操作培训
4. 培训合格后，安排上岗

#### 4.3.2 排班流程

```
制定排班计划 → 发布排班 → 员工查看排班 → 执行排班 → 调整排班
```

**操作规范**：
1. 根据业务需求，制定排班计划
2. 发布排班计划，通知员工
3. 员工查看排班，确认工作时间
4. 按照排班执行工作
5. 如需调整，提前申请
