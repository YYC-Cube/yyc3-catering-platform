#### 3.7 升级指南

##### 3.7.1 从0.9.0升级到1.0.0

**前置条件**
- 备份现有数据库
- 备份配置文件
- 停止运行中的服务
- 确保系统满足1.0.0版本要求

**升级步骤**

1. **代码更新**
   ```bash
   # 拉取最新代码
   git pull origin main
   
   # 切换到1.0.0版本
   git checkout v1.0.0
   
   # 更新依赖
   pnpm install
   ```

2. **数据库迁移**
   ```bash
   # 执行数据库迁移
   pnpm db:migrate
   
   # 验证迁移结果
   pnpm db:migrate:status
   ```

3. **配置更新**
   - 更新环境变量配置
   - 检查配置文件格式
   - 验证配置有效性

4. **构建部署**
   ```bash
   # 构建项目
   pnpm build
   
   # 部署到生产环境
   pnpm deploy:prod
   ```

5. **验证测试**
   - 执行冒烟测试
   - 验证核心功能
   - 检查日志输出
   - 监控系统性能

**回滚方案**
   ```bash
   # 停止新版本服务
   pnpm docker:down
   
   # 切换回旧版本
   git checkout v0.9.0
   
   # 恢复数据库
   pnpm db:restore
   
   # 启动旧版本服务
   pnpm docker:up
   ```
