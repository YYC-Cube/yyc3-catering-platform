# 构建可复用的企业管理知识库体系UI系统设计提示词（全量优化版）

## 一、核心设计目标与原则

**五高标准：**

- **高一致性**：所有组件遵循统一色彩映射规则，模块主题色贯穿全局
- **高可识别性**：通过边线阴影系统建立直观的模块视觉识别体系
- **高响应性**：交互状态具有明确、连贯的视觉反馈层次
- **高可用性**：拒绝纯黑色，采用主题色深变体确保可访问性
- **高复用性**：组件设计标准化，支持跨模块、跨场景复用

**五标规范：**

- **色彩标准**：模块主题色 → 边线阴影色 → 文字/图标色 → 背景浅色
- **交互标准**：静态 → 悬停 → 选中 → 激活四态分明
- **动效标准**：位移范围≤3px，旋转角度≤30°，时长200-400ms
- **层级标准**：边线阴影宽度（菜单3px/卡片4px/激活5px）区分层次
- **状态标准**：滚动透明度80%，展开/收缩保持核心视觉标识

**五化机制：**

- **模块化**：每个功能模块对应一套完整色彩映射方案
- **系统化**：导航、组件、交互形成闭环设计语言
- **动态化**：色彩、阴影、动效随状态、操作、滚动实时响应
- **情感化**：通过微动效与色彩心理学增强操作愉悦感
- **场景化**：适配知识库管理、监控、数据分析等多业务场景

## 二、全局设计系统规范

### 1. 色彩体系（拒绝纯黑色方案）

```
主题色映射规则：
- CPU模块：主题色#2A6EBB → 阴影色#1E5CA3 → 深色文本#0D3A6A → 浅背景#F0F7FF
- 内存模块：主题色#27AE60 → 阴影色#1E8449 → 深色文本#0E5C2F → 浅背景#F0F9F4
- 存储模块：主题色#8E44AD → 阴影色#703688 → 深色文本#4A235A → 浅背景#F8F0FF
- 网络模块：主题色#E67E22 → 阴影色#C15811 → 深色文本#8B3A0A → 浅背景#FFF5EB
- 安全模块：主题色#E74C3C → 阴影色#C0392B → 深色文本#922B21 → 浅背景#FFF0EE
```

### 2. 导航与菜单系统（全状态覆盖）

**A. 基础结构：**

- 单栏文本框布局，右侧固定3px垂直阴影线（色值=模块主题色）
- 高度统一48px，圆角6px，内边距12px

**B. 状态机设计：**

```
1. 静态态：
   - 阴影：3px solid [module-shadow-color]
   - 图标：正常色（主题色80%透明度）
   - 文本：模块深色调

2. 悬停态（:hover）：
   - 保留3px阴影
   - 容器：Y轴位移+1px（缓动ease-out，200ms）
   - 图标：顺时针旋转15°（缓动ease-in-out，300ms）

3. 选中态（.active）：
   - 保留3px阴影
   - 容器：scale(1.02) + 轻微背景色加深（+5%明度）
   - 图标：持续30°旋转（保持状态）

4. 展开/收缩逻辑：
   - 展开：图标+文本，完整阴影显示
   - 收缩：仅图标，图标容器右侧保留3px阴影
   - 过渡：width变化（300ms ease-in-out）

5. 滚动适配：
   - 滚动中：阴影透明度→80%，背景模糊2px
   - 停止后：300ms内恢复100%透明度
```

### 3. 主区域组件库规范

**A. 面包屑导航：**

- 左侧3px垂直阴影（色值=模块阴影色）
- 当前页文字颜色 = 模块主题色
- 分隔符颜色 = 模块主题色30%透明度
- 悬停项：文字加粗 + 背景色（主题色10%透明度）

**B. 搜索框组件：**

- 左右两侧各3px垂直阴影（色值=模块阴影色）
- 输入文字颜色 = 模块深色调（如蓝色模块用#0D3A6A）
- 占位符颜色 = 模块主题色40%透明度
- 搜索图标颜色 = 模块主题色
- 聚焦状态：阴影宽度增至4px + 轻微外发光（主题色20%）

**C. 卡片组件系统：**

- 层级1（内容卡片）：左侧4px阴影 + 浅背景（主题色5%透明度）
- 层级2（数据卡片）：左侧4px阴影 + 渐变背景（浅色→极浅）
- 悬停效果：Y轴位移-2px + 右下3D阴影（2px模糊，主题色20%）
- 点击状态：阴影宽度增至5px + 背景色加深（+8%明度）

**D. 按钮体系：**

```
1. 主要按钮：
   - 边框：1px solid [module-shadow-color]
   - 背景：主题色（80%透明度）
   - 图标：纯白或极浅色调
   - 点击态：边框阴影加粗至5px + 背景全不透明

2. 次要按钮：
   - 仅边框阴影（无填充）
   - 悬停：填充主题色10%透明度
   
3. 图标按钮：
   - 圆形/方形，图标色=主题色
   - 悬停：背景主题色15%透明度
```

**E. 表单控件：**

1. **复选框：**
   - 边框：2px solid [module-shadow-color]
   - 选中：填充色=主题色20%透明度 + ✓颜色=主题色
   - 悬停：边框阴影轻微外扩

2. **进度指示器：**
   - 轨道：浅灰色背景 + 边框（模块阴影色）
   - 进度条：主题色线性渐变
   - 文字标签：主题色深变体
   - 高值警告（>80%）：进度条变深色+脉动动画（2%透明度变化）

**F. 数据看板：**

- 主数值：主题色 + 字体加粗 + 轻微阴影
- 辅助文字：主题色浅变体（60%透明度）
- 趋势图标：主题色，上升/下降对应不同旋转角度
- 卡片背景：极浅主题色调（2-3%透明度）

## 三、交互反馈系统

### 1. 3D阴影层次规则

- **基础层**：所有组件静态右下1px阴影（主题色10%）
- **交互层**：悬停/点击时增加右下3D阴影（2px模糊，主题色20%）
- **激活层**：选中态额外添加内阴影（inset 0 0 0 1px 主题色15%）

### 2. 动效协调体系

```
- 微交互：200-300ms（位移、旋转）
- 状态切换：300-400ms（缩放、透明度）
- 连续动画：500ms+（如进度条加载）
- 缓动函数：交互反馈用ease-out，状态进入用ease-in-out
```

### 3. 滚动行为适配

- 快速滚动：阴影透明度线性降低至60%
- 慢速滚动：阴影透明度保持80%
- 滚动停止：分阶段恢复（100ms至80%，300ms至100%）

## 四、响应式与无障碍设计

### 1. 断点适配

- 移动端（<768px）：菜单收缩为图标模式，阴影保留
- 平板端（768-1024px）：组件边距调整，阴影宽度减1px
- 桌面端（>1024px）：完整实现所有规范

### 2. 无障碍保障

- 色彩对比度：文本与背景至少4.5:1
- 焦点环：使用主题色阴影代替默认蓝色轮廓
- 屏幕阅读器：ARIA标签与状态同步更新
- 键盘导航：Tab顺序与视觉层级一致

## 五、实施提示要点

**生成要求：**

1. 组件覆盖度≥90%，确保知识库所有页面元素有据可循
2. 色彩一致性检测：随机抽查三个模块，验证映射链完整性
3. 交互连贯性：状态切换需平滑，无突变跳转
4. 性能优化：阴影使用CSS变量，避免重复计算
5. 扩展预留：新增模块可通过扩展色彩映射快速接入

**验证清单：**

- [ ] 无任何纯黑色（#000000）元素
- [ ] 所有边线阴影与模块主题色匹配
- [ ] 四态交互（静态、悬停、选中、激活）完整实现
- [ ] 滚动行为与阴影透明度联动正常
- [ ] 移动端收缩模式保持核心视觉标识
- [ ] 色彩对比度满足WCAG 2.1 AA标准
